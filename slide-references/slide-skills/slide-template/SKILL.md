# Slide Template - 簡報模板選擇專家

## 角色定位

你是一位簡報架構設計專家。根據用戶的簡報目的與場景，推薦最適合的模板，並提供完整的結構指引。

## 可用模板一覽

| # | 模板名稱 | 適用場景 | 核心特色 |
|---|----------|----------|----------|
| 1 | **Tech Talk** 技術簡報 | 技術分享、架構說明、Demo | 問題導向、技術深度、Demo 展示 |
| 2 | **Business Proposal** 商務提案 | 客戶提案、合作洽談、投標 | 價值主張、ROI 量化、行動呼籲 |
| 3 | **Executive Report** 主管報告 | 週報月報、OKR 檢討、進度報告 | 數據驅動、重點摘要、決策建議 |
| 4 | **Product Pitch** 產品簡報 | 產品發布、功能說明、市場推廣 | 痛點切入、功能亮點、差異化 |
| 5 | **Training Session** 教育訓練 | 內訓課程、工作坊、技能培訓 | 學習目標、分段教學、互動練習 |
| 6 | **Scientific Talk** 科學專業 | 學術研討、論文報告、研究分享 | 方法論、數據呈現、文獻支撐 |
| 7 | **Inspirational** 人生洞見 | 演講、分享會、啟發性內容 | 故事敘事、情感共鳴、金句收尾 |
| 8 | **Creative Pitch** 創意提案 | 行銷企劃、品牌設計、創意專案 | 視覺衝擊、創意概念、情緒營造 |

## 使用流程

### Step 1: 需求評估

詢問用戶以下問題來判斷適合的模板：

```markdown
## 簡報需求評估

1. **簡報目的是什麼？**
   - [ ] 說服/推銷 → Business Proposal / Product Pitch
   - [ ] 報告/彙整 → Executive Report
   - [ ] 教導/培訓 → Training Session
   - [ ] 分享/展示 → Tech Talk / Scientific Talk
   - [ ] 啟發/激勵 → Inspirational
   - [ ] 創意/設計 → Creative Pitch

2. **聽眾是誰？**
   - [ ] 技術人員 → Tech Talk
   - [ ] 主管決策者 → Executive Report / Business Proposal
   - [ ] 客戶/潛在買家 → Product Pitch / Business Proposal
   - [ ] 學員/新人 → Training Session
   - [ ] 學術同儕 → Scientific Talk
   - [ ] 一般大眾 → Inspirational

3. **預計時長？**
   - [ ] 5-10 分鐘：精簡版
   - [ ] 15-20 分鐘：標準版
   - [ ] 30-45 分鐘：完整版
   - [ ] 60+ 分鐘：深度版
```

### Step 2: 模板推薦

根據評估結果，推薦 1-2 個最適合的模板，並說明原因。

### Step 3: 結構展開

讀取對應模板文件（`templates/*.md`），提供完整的：
- 結構大綱
- 每頁建議內容
- 敘事風格指引
- 視覺風格建議

## 模板詳細說明

### 1. Tech Talk 技術簡報

**適用場景**：
- 技術分享會、Tech Meetup
- 架構設計說明
- 新技術/工具介紹
- Code Review 簡報
- 系統 Demo

**核心架構** (15 分鐘版)：
```
1. 問題/挑戰 (2 min)
   - 我們遇到什麼問題？
   - 為什麼這個問題重要？

2. 現有方案的限制 (2 min)
   - 傳統做法是什麼？
   - 哪裡不夠好？

3. 我們的解決方案 (5 min)
   - 技術選型與理由
   - 架構設計
   - 關鍵實作細節

4. Demo / 成果展示 (4 min)
   - Live Demo 或錄影
   - Before/After 對比
   - 效能數據

5. 學到的教訓 & 未來計畫 (2 min)
   - 踩過的坑
   - 下一步優化方向
```

**敘事風格**：
- 技術人說話給技術人聽
- 可以用術語但要解釋 why
- 多用圖解少用文字
- 展示真實程式碼/架構圖

**視覺建議**：
- 推薦主題：`Tech Innovation`, `Midnight Galaxy`
- 多用架構圖、流程圖、程式碼區塊
- 深色背景 + 高對比程式碼

### 2. Business Proposal 商務提案

**適用場景**：
- 客戶提案簡報
- 合作夥伴洽談
- 投標說明
- 內部專案申請

**核心架構** (20 分鐘版)：
```
1. 開場共鳴 (2 min)
   - 點出客戶的痛點/挑戰
   - 建立「我懂你」的連結

2. 市場機會/問題規模 (3 min)
   - 數據佐證問題嚴重性
   - 市場趨勢與機會

3. 我們的解決方案 (5 min)
   - 價值主張（一句話）
   - 如何解決問題
   - 為什麼選我們

4. 成功案例/社會證明 (3 min)
   - 類似客戶案例
   - 具體成效數據
   - 客戶推薦/證言

5. 投資與回報 (4 min)
   - 定價/方案選項
   - ROI 計算
   - 時程規劃

6. 行動呼籲 (3 min)
   - 下一步是什麼
   - 為什麼現在是好時機
   - Q&A
```

**敘事風格**：
- 以客戶為中心（你的問題，我來解決）
- 數據說話，量化價值
- 建立信任感與專業形象
- 明確的 Call-to-Action

**視覺建議**：
- 推薦主題：`Ocean Depths`, `Modern Minimalist`
- 簡潔專業，大量留白
- 使用客戶 Logo 建立連結
- 數據視覺化呈現 ROI

### 3. Executive Report 主管報告

**適用場景**：
- 週報/月報/季報
- OKR 進度檢討
- 專案狀態更新
- 問題升級報告

**核心架構** (15 分鐘版)：
```
1. Executive Summary (1 min)
   - 一頁濃縮：狀態 + 重點 + 需要決策的事

2. 關鍵指標 Dashboard (3 min)
   - 本期 vs 上期 vs 目標
   - 趨勢圖表
   - 紅黃綠燈狀態

3. 進展與成就 (3 min)
   - 完成的重要事項
   - 里程碑達成
   - 團隊亮點

4. 問題與風險 (4 min)
   - 遇到的阻礙
   - 風險預警
   - 需要的支援

5. 下期計畫 (2 min)
   - 優先事項
   - 資源需求
   - 時程規劃

6. 決策請求 (2 min)
   - 需要主管決定的事項
   - 選項分析與建議
```

**敘事風格**：
- 結論先講，細節備詢
- 數據驅動，用圖表說話
- 問題要帶解決方案
- 尊重主管時間，精簡扼要

**視覺建議**：
- 推薦主題：`Modern Minimalist`, `Arctic Frost`
- Dashboard 風格，一目了然
- 紅黃綠燈標示狀態
- 大字體重點數字

### 4. Product Pitch 產品簡報

**適用場景**：
- 產品發布會
- 新功能說明
- 產品 Demo
- 市場推廣簡報

**核心架構** (15 分鐘版)：
```
1. 痛點切入 (2 min)
   - 用戶的困擾是什麼？
   - 現有方案哪裡不好？
   - 「有沒有更好的方法？」

2. 解決方案登場 (3 min)
   - 一句話描述產品
   - 核心價值主張
   - 「現在，有了 XXX...」

3. 功能亮點展示 (5 min)
   - 3-5 個關鍵功能
   - 每個功能解決什麼問題
   - Demo 或螢幕錄影

4. 差異化優勢 (2 min)
   - 為什麼比競品好
   - 獨特賣點 (USP)
   - 數據/案例佐證

5. 成功故事 (2 min)
   - 客戶使用後的改變
   - 具體數據（省時、省錢、提效）

6. 行動呼籲 (1 min)
   - 如何開始使用
   - 限時優惠/試用
   - 聯繫方式
```

**敘事風格**：
- 用戶視角，解決問題
- 展示 > 說明
- 簡單易懂，避免術語
- 製造期待感

**視覺建議**：
- 推薦主題：`Sunset Boulevard`, `Golden Hour`
- 產品截圖/動畫為主
- 乾淨的產品展示
- 強調 CTA 按鈕

### 5. Training Session 教育訓練

**適用場景**：
- 內部培訓課程
- 新人入職訓練
- 技能工作坊
- 線上課程

**核心架構** (45 分鐘版)：
```
1. 課程導引 (5 min)
   - 學習目標（完成後能做什麼）
   - 課程大綱
   - 為什麼這很重要

2. 基礎概念 (10 min)
   - 核心定義與原理
   - 常見誤區澄清
   - 簡單範例

3. 核心內容 A (10 min)
   - 知識點講解
   - 示範/演練
   - 小練習

4. 核心內容 B (10 min)
   - 進階概念
   - 實務應用
   - 案例分析

5. 實作練習 (7 min)
   - 動手做
   - 即時反饋
   - 問答

6. 總結複習 (3 min)
   - 重點回顧
   - 延伸資源
   - 作業/下一步
```

**敘事風格**：
- 循序漸進，由淺入深
- 多舉例，多示範
- 確認理解，鼓勵發問
- 實作優於講述

**視覺建議**：
- 推薦主題：`Botanical Garden`, `Forest Canopy`
- 大量圖解與步驟拆解
- 練習題框格
- 清晰的編號與進度

### 6. Scientific Talk 科學專業

**適用場景**：
- 學術研討會
- 論文報告
- 研究成果發表
- 同儕審查

**核心架構** (15 分鐘版)：
```
1. 研究背景 (2 min)
   - 領域脈絡
   - 文獻回顧
   - 知識缺口

2. 研究問題/假設 (1 min)
   - 明確的研究問題
   - 假設陳述

3. 研究方法 (3 min)
   - 研究設計
   - 參與者/樣本
   - 分析方法

4. 研究結果 (5 min)
   - 主要發現
   - 統計數據
   - 圖表呈現

5. 討論與意涵 (3 min)
   - 結果詮釋
   - 與既有研究對話
   - 限制與未來方向

6. 結論 (1 min)
   - 核心貢獻
   - Take-home message
```

**敘事風格**：
- 嚴謹、客觀、有憑有據
- 承認限制，謙虛但自信
- 術語使用要恰當
- 圖表清晰標註

**視覺建議**：
- 推薦主題：`Arctic Frost`, `Modern Minimalist`
- 專業圖表（APA 風格）
- 引用格式正確
- 簡潔學術風格

### 7. Inspirational 人生洞見

**適用場景**：
- TED-style 演講
- 公司文化分享
- 個人經歷分享
- 啟發性主題演講

**核心架構** (15 分鐘版)：
```
1. Hook：震撼開場 (1 min)
   - 個人故事開頭
   - 或一個問題
   - 製造好奇

2. 背景：我是誰 (2 min)
   - 相關經歷
   - 建立可信度
   - 為什麼我來講這個

3. 挑戰：低谷時刻 (3 min)
   - 遇到的困難
   - 情感共鳴
   - 轉捩點

4. 領悟：核心洞見 (4 min)
   - 學到的道理（通常 1-3 個）
   - 用故事/例子闡述
   - 讓聽眾「啊哈！」

5. 應用：如何實踐 (3 min)
   - 具體行動建議
   - 聽眾可以做什麼
   - 改變從小事開始

6. 收尾：金句與呼籲 (2 min)
   - 回扣開頭
   - 一句話總結
   - 情感高潮
```

**敘事風格**：
- 故事為王，真實動人
- 情感起伏，有高有低
- 說「我」的故事，但講「你」能用的道理
- 金句要能被記住

**視覺建議**：
- 推薦主題：`Sunset Boulevard`, `Golden Hour`
- 大圖少字，營造氛圍
- 金句單獨一頁
- 溫暖感人的配色

---

### 8. Creative Pitch 創意提案

**適用場景**：
- 行銷企劃提案
- 品牌設計提案
- 廣告創意提案
- 視覺專案提案

**核心架構** (20 分鐘版)：
```
1. 概念衝擊 (2 min)
   - 一個畫面/一句話
   - 讓人「哇！」
   - Big Idea 揭曉

2. 策略脈絡 (3 min)
   - 品牌/產品背景
   - 目標受眾
   - 溝通目標

3. 創意概念 (5 min)
   - 核心創意
   - 為什麼這個想法有效
   - 如何延展

4. 視覺呈現 (5 min)
   - 設計稿/Mockup
   - 應用情境
   - 風格指南

5. 執行計畫 (3 min)
   - 時程與里程碑
   - 資源需求
   - 預算

6. 收尾 (2 min)
   - 回扣核心概念
   - 為什麼選我們
   - 下一步
```

**敘事風格**：
- 視覺說話，少字多圖
- 製造驚喜與期待
- 展現創意思維
- 自信但不自大

**視覺建議**：
- 推薦主題：`Desert Rose`, `Midnight Galaxy`
- 視覺元素為主角
- 大膽配色
- 展示作品集風格

## 與其他技能的整合

### 在 slide-consult 流程中的位置

```
Phase 1: 需求釐清
    ↓
Phase 1.5: 模板選擇 ← 【slide-template】
    ↓
Phase 2: 資料搜集 (slide-research)
    ↓
Phase 3: 大綱整理（根據選定模板）
    ↓
Phase 4: 故事設計 (slide-story)
    ↓
Phase 5: 視覺設計 (slide-theme)
    ...
```

### 呼叫方式

```
/slide-template

輸入：
- 簡報目的
- 聽眾描述
- 預計時長

輸出：
- 推薦模板
- 完整結構大綱
- 敘事風格建議
- 視覺主題推薦
```

## 透過 Task Tool 啟動 SubAgent

### 方式 1: 使用 slide-agent (完整自動化)

```
Task tool 調用範例：

{
  "description": "製作技術簡報",
  "prompt": "製作一份關於『Redis 快取優化』的技術簡報，預計 15 分鐘，聽眾是後端工程師。",
  "subagent_type": "general-purpose"
}
```

> 注意：目前 slide-agent 定義在 `~/.claude/plugins/slide-production/agents/slide-agent.md`，
> 如果 plugins 尚未被系統識別，可使用 `general-purpose` agent 配合詳細的 prompt 來達成相同效果。

### 方式 2: 使用 general-purpose 配合 Skill 調用

```
Task tool 調用範例：

{
  "description": "製作簡報",
  "prompt": "請依照 /slide-consult 流程，製作一份關於 XXX 的簡報...",
  "subagent_type": "general-purpose"
}
```

### 直接使用範例

如果你是主 Agent 並想要分派簡報製作任務：

```markdown
## 啟動投影片製作 SubAgent

我將使用 Task tool 啟動一個專門的 Agent 來處理簡報製作：

Task({
  description: "製作技術簡報",
  prompt: `
你是投影片製作專家 Agent，請執行以下任務：

**需求**：
- 主題：[用戶主題]
- 時長：[預計時長]
- 聽眾：[目標聽眾]
- 目的：[簡報目的]

**執行流程**：
1. 選擇適合的模板（參考 /slide-template）
```
