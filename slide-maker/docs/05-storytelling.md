# 故事設計框架

## 概述

投影片不只是資訊的堆疊，而是一場有結構的敘事。故事設計框架定義了如何將原始素材轉化為有說服力的簡報敘事，涵蓋三大支柱、風格光譜、敘事結構、視覺化技法與文案風格。

---

## 三大支柱

每份有效的簡報都建立在三根支柱上：

### Data（證據）

用數據和事實支撐論點。

```
✅ "導入自動化後，部署頻率從每月 1 次提升到每週 3 次"
❌ "導入自動化後，效率大幅提升"
```

💡 數據不一定是數字，也可以是案例、引用、研究結果。關鍵是可驗證。

### Narrative（意義）

替數據賦予脈絡，讓聽眾理解「所以呢？」。

```
✅ Data: 客訴量下降 40%
   Narrative: "這代表我們的客戶終於不用在週五晚上打電話罵人了"

❌ Data: 客訴量下降 40%
   Narrative: "客訴量有顯著改善"（只是重複數據）
```

### Visuals（清晰）

用視覺元素降低理解成本。

```
✅ 用流程圖呈現 5 步驟的部署流程
❌ 用 5 段文字描述同一個流程
```

⚠️ 三根支柱缺一不可。只有 Data 會枯燥，只有 Narrative 會空洞，只有 Visuals 會華而不實。

## 風格光譜

### 不是所有簡報都需要煽情

風格光譜從理性到感性，依據目的與受眾選擇定位：

```
理性/客觀 ◄─────────────────────────────► 感性/說服

技術報告    架構提案    商務簡報    產品發表    激勵演講
  │           │          │          │          │
  ▼           ▼          ▼          ▼          ▼
 數據為主   邏輯推演   利益導向   願景描繪   情感連結
```

| 定位 | Data 佔比 | Narrative 佔比 | 語氣 | 適用場景 |
|------|----------|---------------|------|---------|
| 理性客觀 | 70% | 20% | 精確、中性 | Code Review、技術分享 |
| 專業分析 | 50% | 35% | 專業、有條理 | 架構設計提案 |
| 商務說服 | 40% | 45% | 自信、有力 | 客戶提案、預算申請 |
| 願景導向 | 25% | 55% | 前瞻、興奮 | 產品發表、募資 |
| 情感驅動 | 15% | 65% | 真誠、有溫度 | 團隊激勵、年度回顧 |

💡 選錯風格比內容不好更致命。對工程師用煽情語氣會被翻白眼，對投資人用純技術語言會失去注意力。

## 敘事結構

### 結構一：Problem-Solution（問題解決型）

```
痛點 → 影響 → 解法 → 成果

Slide 1: [痛點] 目前手動部署平均耗時 4 小時
Slide 2: [影響] 每月浪費 80 人時，且出錯率 15%
Slide 3: [解法] 導入 CI/CD Pipeline 自動化部署
Slide 4: [成果] 部署時間降到 15 分鐘，出錯率 < 1%
```

✅ 適用於：提案、改善報告、新工具導入
⚠️ 注意：痛點必須是聽眾真正在意的，不是你覺得重要的

### 結構二：Trend（趨勢演變型）

```
過去 → 現在 → 未來

Slide 1: [過去] 2020 年我們還在用 FTP 上傳程式碼
Slide 2: [現在] 2024 年已全面容器化，K8s 管理 200+ 服務
Slide 3: [未來] 2026 年目標：Serverless-first 架構
```

✅ 適用於：年度回顧、技術演進、市場分析
⚠️ 注意：未來的部分要有依據，不能只是願望

### 結構三：Comparison（比較型）

```
A 方案 vs B 方案 → 明確勝出者

Slide 1: [A 方案] 自建機房：初期成本 $500K，維護 3 人
Slide 2: [B 方案] 雲端遷移：初期成本 $50K，彈性擴展
Slide 3: [對比] 3 年 TCO 比較表
Slide 4: [結論] B 方案在成本與彈性上全面勝出
```

✅ 適用於：技術選型、供應商評估、架構決策
⚠️ 注意：對比要公平，刻意偏頗會失去信任

### 混合結構

```
範例：新系統提案（30 分鐘）

Part 1 - Trend（3 slides）: 產業趨勢與技術演進
Part 2 - Problem-Solution（5 slides）: 現有痛點與解法
Part 3 - Comparison（4 slides）: 方案比較與推薦
Part 4 - 結論與 Call to Action（2 slides）
```

## 視覺化技法

### Progressive Reveal（漸進揭露）

一次只展示一個重點，避免資訊過載。

```
❌ 一張投影片塞 5 個重點，聽眾不知道看哪裡

✅ 正確做法：
   Step 1: 只顯示第一個重點
   Step 2: 加入第二個重點（第一個淡化）
   Step 3: 加入第三個重點
```

### Contrast（對比強調）

用視覺差異突顯關鍵資訊。

```
✅ Before/After 並排對比、關鍵數字放大 3 倍
❌ 所有元素同樣大小、同樣顏色 → 什麼都不突出
```

### Annotation（標註引導）

在圖表或截圖上加入標註，引導視線。

```
✅ 在架構圖上用箭頭標記「瓶頸在這裡」
✅ 在折線圖上標記關鍵轉折點
❌ 放一張原始截圖，沒有任何標記
```

⚠️ 標註不要超過 3 個，太多等於沒有

## 文案風格指引

### 技術分享

語氣：理性、客觀、精確

```
✅ "此架構採用 Event Sourcing Pattern，
    事件存儲使用 EventStoreDB，
    讀取模型透過 Projection 同步至 PostgreSQL"
❌ "我們選了一個超棒的架構來解決所有問題！"
```

### 商務提案

語氣：專業、有說服力、以利益為導向

```
✅ "導入此方案預估每年節省 $240K 人力成本，
    ROI 在第 8 個月達到損益平衡"
❌ "這個方案會讓公司賺很多錢"
```

### 激勵演講

語氣：感性、有故事性、真誠

```
✅ "去年這個時候，我們的系統每週當機 3 次。
    今天站在這裡，我們已經連續 180 天零事故。"
❌ "我們的系統穩定性有了顯著的全方位提升"
```

## 情緒曲線設計

### 能量等級模型

每張投影片都有一個情緒能量等級（1-5）：

```
能量 5 │         ★ 高潮
       │        / \
能量 4 │       /   \      ★ Call to Action
       │      /     \    /
能量 3 │     /       \  /
       │    /         \/
能量 2 │   / 鋪陳
       │  /
能量 1 │ ★ 開場
       └──────────────────────── 時間
```

### 典型 15 分鐘簡報的情緒設計

| 段落 | 投影片 | 能量 | 目的 |
|------|--------|------|------|
| 開場 | 1-2 | 2→3 | 建立連結，拋出問題 |
| 痛點 | 3-4 | 3→2 | 讓聽眾感受問題的嚴重性 |
| 轉折 | 5 | 2→4 | 「但如果有更好的方法呢？」 |
| 解法 | 6-8 | 4 | 展示方案，維持高能量 |
| 證據 | 9-10 | 3→4 | 用數據強化信心 |
| 高潮 | 11 | 5 | 最強的一句話或數據 |
| 收尾 | 12-13 | 4→3 | Call to Action |

### 能量控制技巧

```
提升能量：放大關鍵數字、高對比色彩、一張投影片只放一句話
降低能量：柔和色調、展示細節數據表、多文字少圖片

⚠️ 不要全程高能量（聽眾疲勞）或全程低能量（聽眾睡著）
💡 關鍵是起伏節奏，像呼吸一樣
```

## 快速檢查清單

設計前：

- [ ] 確認受眾是誰？選定風格定位（理性 ↔ 感性）
- [ ] 選定敘事結構（Problem-Solution / Trend / Comparison）
- [ ] 三大支柱是否平衡？Data + Narrative + Visuals

設計後：

- [ ] 情緒曲線有起伏嗎？
- [ ] 每張投影片只有一個核心訊息？
- [ ] 文案風格與受眾匹配？
