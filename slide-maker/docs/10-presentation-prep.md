# 演講準備

## 概述

簡報做完只是一半，另一半是「講好它」。本文件涵蓋兩大準備工作：演講逐字稿撰寫（Part A）與 Q&A 預備（Part B）。

---

# Part A：演講逐字稿

## 三大核心原則

### 原則一：口語化

逐字稿是寫給嘴巴的，不是寫給眼睛的。

| ❌ 書面語 | ✅ 口語 |
|----------|--------|
| 「根據第三季財報數據顯示」 | 「我們來看 Q3 的數字」 |
| 「此方案具備高度可行性」 | 「這個方案做得到」 |
| 「綜合以上各項分析」 | 「所以回到重點」 |

💡 測試方法：把稿子唸出來，如果聽起來像在讀公文，就需要改。

### 原則二：有節奏、有呼吸

```
「去年我們的客戶滿意度是 72 分。[PAUSE]
今年 [SLOW] 我們做到了 91 分。[LONG PAUSE]
這不是靠運氣。[EMPHASIS] 這是靠每一位同仁 [GESTURE] 的堅持。」
```

### 原則三：情緒起伏

```
能量  9 │      ★               ★★
      8 │    ★   ★     ★ ★  ★★
      7 │  ★       ★ ★     ★
      6 │★           ★   ★
        └──────────────────────→ 時間
         開場  問題 方案 證據 收尾
```

- 開場：能量 7-8（抓注意力）
- 鋪陳問題：能量 6-7（讓聽眾感受痛點）
- 提出方案：能量 8-9（高潮）
- 收尾 CTA：能量 8-9（推向行動）

## 標記系統

### 停頓與節奏

| 標記 | 意義 | 使用時機 |
|------|------|---------|
| `[PAUSE]` | 停頓 1-2 秒 | 重點前後、讓聽眾消化 |
| `[LONG PAUSE]` | 停頓 3-5 秒 | 重大數據揭曉、情緒轉折 |
| `[SLOW]` | 放慢語速 | 強調關鍵數字或結論 |

### 語氣與動作

| 標記 | 意義 | 使用時機 |
|------|------|---------|
| `[EMPHASIS]` | 加重語氣 | 核心論點、關鍵字 |
| `[SOFT]` | 放輕聲音 | 故事、情感段落 |
| `[LOOK UP]` | 抬頭看觀眾 | 建立眼神連結 |
| `[GESTURE]` | 手勢輔助 | 指向投影片、比劃數據 |

### 操作與互動

| 標記 | 意義 | 使用時機 |
|------|------|---------|
| `[CLICK]` | 切換投影片 | 換頁時機 |
| `[Q]` | 向觀眾提問 | 互動環節 |
| `[HH:MM]` | 時間標記 | 標示預計到達時間點 |

### 標記使用範例

```markdown
[00:00] [CLICK]
各位早安。[PAUSE]
我今天要分享的主題，跟在座每個人都有關。[LOOK UP]
[Q] 你上次覺得開會有收穫，是什麼時候？[LONG PAUSE]
如果你想不太起來 [SOFT]，那很正常。
因為根據我們的調查，[CLICK] [EMPHASIS] 73% 的員工認為超過一半的會議是浪費時間。
```

## 三種敘事弧線

### Problem → Solution（問題解決型）

適用：產品提案、改善方案、技術導入

```
[痛點描述] → [現狀分析] → [方案提出] → [預期效益] → [CTA]
```

### Trend（趨勢分析型）

適用：產業分析、策略規劃、年度展望

```
[趨勢觀察] → [數據佐證] → [對我們的影響] → [建議行動] → [CTA]
```

### Comparison（比較型）

適用：方案評估、A/B 選擇、競品分析

```
[現況] → [方案 A] → [方案 B] → [比較分析] → [推薦] → [CTA]
```

## 語言技巧庫

### 開場 Hook

| 技巧 | 範例 |
|------|------|
| 提問法 | 「你有沒有想過，為什麼我們的部署還要花 3 小時？」 |
| 數據衝擊 | 「去年我們因為系統當機，損失了 1,200 萬。」 |
| 故事法 | 「上個月有個客戶打電話來，他說了一句話讓我們整個團隊沉默。」 |
| 反差法 | 「大家都說 AI 會取代工程師。但今天我要說的剛好相反。」 |

⚠️ 開場絕對不要用「大家好，我是 XXX，今天要報告的主題是...」這種方式。

### 轉場與情緒調節

| 技巧 | 範例 |
|------|------|
| 承上啟下 | 「知道了問題，接下來看我們怎麼解。」 |
| 預告 | 「接下來這張圖，是今天最重要的一張。」 |
| 回扣 | 「還記得剛才提到的 73% 嗎？看看現在的數字。」 |

## 五階段練習計畫

| 階段 | 方式 | 重點 |
|------|------|------|
| 1. 默讀 | 看稿默讀一遍 | 熟悉內容與流程 |
| 2. 朗讀 | 大聲念出來 (2 次) | 抓口語節奏，修改拗口處 |
| 3. 計時 | 對著鏡子講 (2 次) | 控制時間、練習手勢表情 |
| 4. 錄影 | 錄下來自己看 | 找出口頭禪、不自然的動作 |
| 5. 模擬 | 找人當觀眾講一次 | 接受回饋、練習應對突發狀況 |

💡 至少完成到階段 3 才能上台。

# Part B：Q&A 準備

## 問題分類

| 類型 | 定義 | 範例 | 應對重點 |
|------|------|------|---------|
| Clarification | 聽不懂，想確認 | 「轉換率是指什麼？」 | 簡短重述 |
| Deep Dive | 想知道更多 | 「技術架構可以再說明嗎？」 | 備用投影片 |
| Challenge | 質疑論點 | 「樣本數夠嗎？」 | 用數據回應 |
| Practical | 關心執行面 | 「需要多少預算？」 | 準備具體數字 |
| Hostile | 帶有攻擊性 | 「上次你們也這樣說」 | 先同理再回應 |

## 受眾角色預測

| 角色 | 關心的事 | 準備方向 |
|------|---------|---------|
| 決策者 (Decision-maker) | ROI、風險、時程 | 準備財務數據 |
| 技術專家 (Technical Expert) | 架構、可行性 | 準備技術比較表 |
| 財務人員 (Finance) | 成本、預算 | 準備 TCO 分析 |
| 用戶代表 (User Representative) | 易用性、影響 | 準備使用情境說明 |
| 質疑者 (Skeptic) | 證據、過往紀錄 | 準備 Case Study |

## 問題發生機率

| 機率 | 標記 | 準備深度 |
|------|------|---------|
| 高 | 🔴 | 完整回答 + 備用數據 |
| 中 | 🟡 | 2-3 句重點回答 |
| 低 | 🟢 | 心裡有個方向就好 |

## PREP 回答結構

```
P - Point（結論）：先講結論
R - Reason（原因）：為什麼這麼說
E - Example（例子）：舉證或案例
P - Point（重申）：重複結論
```

**範例：**

```
Q: 「為什麼不用雲端方案？」
P: 「我們評估後認為混合架構更適合。」
R: 「因為部分資料受法規限制，必須放在地端。」
E: 「像是員工個資和財務資料，依照個資法不能放境外。」
P: 「所以混合架構是兼顧效能與合規的最佳選擇。」
```

## Bridge 技巧：處理離題問題

```
1. 肯定問題（不要說「這跟主題無關」）
2. 搭橋（轉回相關主題）
3. 回到重點

範例：
Q:「你們有沒有考慮進軍東南亞市場？」
「這是很好的問題。東南亞市場確實是公司的長期目標之一。
 而今天提案的這套系統，其實就是為了未來國際化做準備。」
```

## 處理「我不知道」的情境

✅ 正確：「我手邊沒有確切數字，會後 24 小時內 email 給您。不過就我目前了解...」

❌ 錯誤：硬掰答案、說「我不知道」就停住、明顯閃躲

## 敵意問題應對

```
Step 1: 深呼吸 [PAUSE]（不要立即反應）
Step 2: 同理 —「我理解您的顧慮」
Step 3: 重新框架 — 把攻擊轉為建設性討論
Step 4: 用事實回應 — 不要用情緒對抗情緒
Step 5: 邀請後續討論 —「我們可以會後再深入聊」
```

⚠️ 絕對不要在台上跟人吵架。你贏了辯論，但輸了整場演講。

## Q&A 通用注意事項

- ✅ 重複問題再回答（確保全場都聽到）
- ✅ 回答控制在 60-90 秒
- ✅ 回答完問「這樣有回答到您的問題嗎？」
- ❌ 不要回答超過 2 分鐘
- ❌ 不要只看提問者，要看全場
- ❌ 不要用「好問題！」回答每個問題（會顯得敷衍）
