# 快速入門（5 分鐘）

> 從零開始製作一份簡報的最快路徑

---

## 方法一：一句話啟動（推薦）

直接告訴 AI 你的需求：

```
/slide-consult

我要做一份 15 分鐘的技術分享簡報，主題是「微服務架構遷移實戰」，
聽眾是公司的後端工程師團隊。
```

系統會自動：
1. 釐清需求 → 2. 選擇模板 → 3. 搜集資料 → 4. 設計故事 → 5. 製作投影片 → 6. 品質審核

---

## 方法二：分步驟操作

### Step 1: 選擇模板

```
/slide-template

我要做技術分享，聽眾是工程師，約 15 分鐘。
```

系統推薦 **Tech Talk** 模板，提供結構大綱。

### Step 2: 搜集資料

```
/slide-research

主題：微服務架構遷移
需要：業界案例、技術比較、最佳實踐
```

產出研究報告，含引用來源。

### Step 3: 設計故事線

```
/slide-story

用 Problem-Solution 框架：
- 痛點：單體架構的瓶頸
- 解法：微服務遷移策略
- 成果：效能提升數據
```

### Step 4: 製作投影片

```
/slide-maker

根據上面的大綱和研究資料，製作 15 頁投影片。
使用 Tech Innovation 主題。
```

### Step 5: 品質審核

```
/slide-reviewer

審核剛才製作的投影片，特別注意技術深度和邏輯性。
```

### Step 6: 準備演講

```
/slide-script  → 生成演講逐字稿
/slide-qa      → 準備 Q&A 回答
/slide-export  → 匯出完整套件
```

## 特殊場景快速啟動

### MAYO 品牌簡報

```
/slide-consult

我要做 MAYO 品牌風格的產品簡報。
```

系統自動載入 `/mayo-slide-expert` 進行品牌審核。

### 向上匯報

```
/slide-consult

我要做給總經理的月報簡報。
```

系統自動載入 `/exec-report-coach`，確保每頁有目的、訴求、行動。

### 英文教學繪本

```
/slide-consult

我要做 5 歲小朋友的英文互動繪本，主題是「三隻小豬」。
```

系統自動整合 `english-teaching-expert` 團隊。
