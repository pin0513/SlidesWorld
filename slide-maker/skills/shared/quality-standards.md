# Slide Maker 共用品質標準

> 本文件定義所有 slide-maker skills 共用的品質標準。
> 各 skill 應在各自的檢查流程中引用此標準。

---

## 1. 視覺設計標準 (Visual Standards)

### 字體大小

| 元素 | 最小值 | 建議值 | 說明 |
|------|--------|--------|------|
| 標題 (Title) | 36pt | 40-48pt | 主標題，一眼可辨識 |
| 副標題 (Subtitle) | 28pt | 30-36pt | 輔助標題 |
| 內文 (Body) | 28pt | 28-32pt | 條列文字、段落 |
| 圖表標籤 | 18pt | 20-24pt | 圖表中的文字 |
| 註腳 (Footnote) | 14pt | 14-16pt | 來源、備註 |

**規則**: 投影片上任何文字不得小於 14pt，主要內容不得小於 28pt。

### 留白與排版

| 指標 | 標準 | 說明 |
|------|------|------|
| 留白比例 | >= 40% | 投影片面積中至少 40% 為空白 |
| 邊距 (Margin) | >= 5% | 四邊各留至少 5% 的 padding |
| 行距 (Line Height) | 1.4-1.6 | 文字行距 |
| 段落間距 | 0.8-1.2em | 段落之間的間距 |
| 最大文字行數 | 6 行 | 單頁投影片文字上限 |
| 最大條列項目 | 5 項 | 單頁 bullet points 上限 |

### 圖片品質

| 指標 | 標準 | 說明 |
|------|------|------|
| 解析度 | >= 1024x1024 | AI 生成圖片最低解析度 |
| 投影片中圖片 | >= 800px 寬 | 投影片中使用的圖片 |
| 圖片壓縮 | WebP/PNG | 優先使用無損或高品質格式 |
| 圖片佔比 | 40-70% | 圖文並茂時，圖片佔投影片面積比 |
| Alt Text | 必要 | 所有圖片需有替代文字 |

### 色彩與對比

| 指標 | 標準 | 說明 |
|------|------|------|
| 文字/背景對比度 | >= 4.5:1 | WCAG AA 標準 |
| 大文字對比度 | >= 3:1 | 24pt 以上文字 |
| 主色數量 | 2-3 色 | 主要配色不超過 3 色 |
| 強調色 | 1 色 | 用於 CTA 或重點標記 |
| 漸層使用 | 謹慎 | 避免影響文字可讀性 |

### 動畫與轉場

| 指標 | 標準 | 說明 |
|------|------|------|
| 轉場效果 | Fade/Slide | 僅使用簡潔轉場 |
| 動畫時長 | 0.3-0.5s | CSS transition duration |
| 進場動畫 | 最多 1 種/頁 | 避免過多動畫干擾 |
| 自動播放 | 禁止 | 不使用自動播放動畫 |

## 2. 內容品質標準 (Content Standards)

### 核心原則

| 原則 | 說明 | 檢查方式 |
|------|------|----------|
| One Message Per Slide | 每頁只傳達一個核心訊息 | 能否用一句話總結該頁 |
| Evidence-Based | 論點需有數據或案例支撐 | 檢查是否有來源標註 |
| Clear CTA | 結尾需有明確行動呼籲 | 聽眾知道下一步該做什麼 |
| Progressive Disclosure | 資訊逐步揭露，避免資訊過載 | 每頁只展示當下需要的資訊 |

### 文字密度

| 投影片類型 | 最大字數 | 說明 |
|-----------|---------|------|
| 標題頁 | 20 字 | 標題 + 副標題 |
| 內容頁 | 60 字 | 含標題與 bullet points |
| 數據頁 | 40 字 | 圖表 + 標註文字 |
| 引言頁 | 30 字 | 引言 + 出處 |
| 結尾頁 | 25 字 | CTA + 聯繫方式 |

### 敘事結構

```
推薦架構：
1. Hook (鉤子)        - 抓住注意力，提出問題
2. Context (脈絡)     - 建立背景，說明為什麼重要
3. Content (內容)     - 核心論點，數據支撐
4. Climax (高潮)      - 關鍵轉折或重要發現
5. Conclusion (結論)  - 總結與行動呼籲
```

### 引用與出處

| 要求 | 說明 |
|------|------|
| 數據必須標註來源 | 「根據 Gartner 2025 報告」 |
| 引言必須標註出處 | 「-- Steve Jobs, 2005」 |
| 統計數字須註明年份 | 避免使用過時數據 |
| 圖片須標註授權 | AI 生成標註「AI Generated」 |

## 3. 行動裝置相容性 (Mobile Compatibility)

### Viewport 設定

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0,
      maximum-scale=1.0, user-scalable=no">
```

### 觸控目標

| 指標 | 標準 | 說明 |
|------|------|------|
| 最小觸控區域 | 44x44px | Apple HIG / Material Design |
| 按鈕間距 | >= 8px | 避免誤觸 |
| 導航箭頭 | >= 48x48px | 左右切換按鈕 |
| 進度指示器 | 可點擊 | 點擊跳轉至特定頁 |

### 行動裝置字體

| 元素 | 最小值 | 說明 |
|------|--------|------|
| 標題 | 20px | 行動裝置上的標題 |
| 內文 | 14px | 行動裝置上任何文字不小於 14px |
| 按鈕文字 | 16px | 按鈕內的文字 |
| 圖表標籤 | 12px | 行動裝置上的圖表 (例外允許) |

### Responsive 斷點

| 斷點 | 寬度 | 說明 |
|------|------|------|
| Desktop | >= 1024px | 標準投影片比例 |
| Tablet | 768-1023px | iPad 等平板 |
| Mobile | < 768px | 手機瀏覽 |

### 行動裝置限制

- 避免 hover 效果作為唯一互動方式
- 確保手勢操作 (swipe) 流暢
- 圖片使用 `srcset` 提供不同解析度
- 避免使用 Flash 或其他不支援的技術

## 4. 時間配置標準 (Timing Standards)

### 每頁時間

| 投影片類型 | 建議時間 | 說明 |
|-----------|---------|------|
| 標題頁 | 30-60 秒 | 快速帶過或等待就座 |
| 內容頁 | 1-2 分鐘 | 標準講解時間 |
| 數據頁 | 1.5-2 分鐘 | 需要時間消化數據 |
| 互動頁 | 2-3 分鐘 | 提問或討論 |
| 結尾頁 | 1-2 分鐘 | CTA 與總結 |

### 整體時間配置

```
總時長分配建議：

┌─────────────────────────────────────────────────────────┐
│ Opening (10-15%)                                        │
│ ├── Hook + 自我介紹 + 議程概覽                            │
├─────────────────────────────────────────────────────────┤
│ Body (65-75%)                                           │
│ ├── 核心內容 (40-50%)                                    │
│ ├── 案例/Demo (15-20%)                                  │
│ └── 數據支撐 (10-15%)                                    │
├─────────────────────────────────────────────────────────┤
│ Closing (10-15%)                                        │
│ ├── 總結重點                                             │
│ ├── CTA                                                 │
│ └── Q&A 預告                                            │
├─────────────────────────────────────────────────────────┤
│ Q&A (10-20%) (如有)                                     │
│ └── 視場合安排                                            │
└─────────────────────────────────────────────────────────┘
```

### 投影片數量建議

| 總時長 | 建議頁數 | 說明 |
|--------|---------|------|
| 5 分鐘 | 5-8 頁 | Lightning Talk |
| 10 分鐘 | 8-12 頁 | 短講 |
| 20 分鐘 | 15-20 頁 | Conference Talk |
| 30 分鐘 | 20-30 頁 | 專題演講 |
| 60 分鐘 | 30-45 頁 | Workshop/Seminar |

**經驗法則**: 平均每頁 1-2 分鐘，圖重頁可快些，數據頁需慢些。

## 5. 語言與用詞標準 (Language Standards)

### 基本原則

| 規則 | 說明 | 範例 |
|------|------|------|
| 使用繁體中文 | 預設使用 Traditional Chinese | 「使用者」非「用户」 |
| 技術名詞保留英文 | 專有名詞不翻譯 | API, SDK, Docker, Kubernetes |
| 避免中英混雜句型 | 英文詞彙嵌入中文句時需自然 | 「透過 API 取得資料」 |
| 簡潔有力 | 投影片文字精簡 | 「提升 30% 效能」非「在效能方面有了約 30% 左右的提升」 |

### 翻譯對照

| English | 繁體中文 | 備註 |
|---------|---------|------|
| User | 使用者 | 非「用戶」 |
| Feature | 功能 | 非「特性」 |
| Performance | 效能 | 非「性能」 |
| Deploy | 部署 | 非「部属」 |
| Architecture | 架構 | 非「架构」 |
| Database | 資料庫 | 非「数据库」 |
| Interface | 介面 | 非「接口」 |
| Program | 程式 | 非「程序」 |
| Information | 資訊 | 非「信息」 |
| Network | 網路 | 非「网络」 |

## 6. 品質檢查清單 (Quality Checklist)

### 提交前必檢項目

#### 視覺設計
- [ ] 所有文字 >= 28pt (內文) 或 >= 14pt (註腳)
- [ ] 留白比例 >= 40%
- [ ] 每頁條列項目 <= 5 項
- [ ] 文字/背景對比度 >= 4.5:1
- [ ] 圖片解析度足夠且有 alt text

#### 內容品質
- [ ] 每頁只傳達一個核心訊息
- [ ] 數據有來源標註
- [ ] 結尾有明確 CTA
- [ ] 文字密度符合標準

#### 行動裝置
- [ ] 觸控目標 >= 44x44px
- [ ] 行動裝置字體 >= 14px
- [ ] 手勢操作流暢
- [ ] Responsive 佈局正常

#### 時間配置
- [ ] 每頁平均 1-2 分鐘
- [ ] 總頁數與演講時間匹配
- [ ] Opening/Body/Closing 比例合理

#### 語言用詞
- [ ] 使用繁體中文
- [ ] 技術名詞保留英文
- [ ] 文字簡潔有力

## 7. 評分標準 (Scoring Rubric)

各 skill 在審查時可使用以下評分標準：

| 維度 | 權重 | 90-100 (優秀) | 70-89 (良好) | < 70 (需改善) |
|------|------|---------------|--------------|---------------|
| 視覺設計 | 25% | 全部符合標準 | 1-2 項未達 | 3+ 項未達 |
| 內容品質 | 30% | 訊息清晰有力 | 部分頁面模糊 | 缺乏焦點 |
| 行動相容 | 15% | 完全 responsive | 部分問題 | 無法使用 |
| 時間配置 | 15% | 節奏流暢 | 稍有不均 | 明顯失衡 |
| 語言用詞 | 15% | 精準簡潔 | 偶有冗贅 | 中英混亂 |

**總分 >= 80 分**: 可交付
**總分 70-79 分**: 需小幅修正
**總分 < 70 分**: 需重大修改
